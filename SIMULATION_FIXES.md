## Context

This is an active inference adaptive learning system for algebra. The core claim is:
- **Meta question (simulation):** Monte Carlo simulation of synthetic students normatively maps optimal curricula without real-world data
- **Normal question (live system):** A POMDP agent minimizes Expected Free Energy to select teaching actions in real time

The simulation is the scientific core. The fixes below address gaps between what the code does and what the research claims.

---

## Fix 1 — Add the Active Inference Policy to the Monte Carlo

**Files:** `simulation/monte_carlo.py`, `active_inference/pomdp.py`

**Problem:** The simulation compares `meta_function` (rule-based), `random`, `fsrs_only`, and `fixed_curriculum`. The actual active inference agent (`ActiveInferenceAgent` using pymdp EFE minimization) is never evaluated in the simulation. The research claim is that EFE beats rule-based — this is currently untested.

**Fix:** Add a fifth policy `"active_inference"` to the `POLICIES` dict in `monte_carlo.py`. This policy:
1. Instantiates `ActiveInferenceAgent` once per trajectory (store it on the agent or pass it through)
2. On each problem, calls `ai_agent.step(correct, response_time_ms, explanation_quality)` to get the next action
3. Maps the returned action name to a skill selection (use the skill with lowest RS as target, same as `_policy_fsrs_only`)

The `ActiveInferenceAgent` already exists in `active_inference/pomdp.py` and is used in `server.py`. Bring it into the simulation.

## Fix 2 — Add Problem Features to the A Matrix

**Files:** `active_inference/pomdp.py`, `active_inference/state_space.py`, `domain/taxonomy.py`

**Problem:** The A matrix is `P(correct | RS, Schema, WM, Affect)`. Problem EI and difficulty are not in the observation model — they're only used to compute WM load as a side effect. This means a tier-1 problem and a tier-7 problem look identical to the POMDP.

**Fix:** Add EI as a conditioning factor in the accuracy A matrix. Concretely:
- Discretize EI into 3 bins: low (1–3), medium (4–6), high (7–9)
- Add this as a new observation modality OR as an additional state factor
- The cleanest approach: add `EI_bin` as a 4th observation modality with 3 levels, and condition the accuracy A matrix jointly on `(RS, Schema, WM, EI_bin)`
- The expertise reversal effect should be encoded: at schema=full, high EI problems become *easier* (schema compresses elements), and worked examples become *less* helpful

Reference `domain/taxonomy.py` `TIER_BASE_EI` and `effective_ei()` for the EI values.

## Fix 3 — Empirically Ground the Student Parameter Distributions

**Files:** `simulation/learner_types.py`

**Problem:** `sample_learner_params()` uses uniform independent distributions. Real student parameters are neither uniform nor independent.

**Fix:** Replace the uniform sampling in `sample_learner_params()` with:

```python
# WM capacity: Normal(4.0, 1.5), clamp to [2, 7]  — Cowan (2001)
wm_capacity = np.clip(rng.normal(4.0, 1.5), 2.0, 7.0)

# Math anxiety: 20% prevalence — Ashcraft (2002)
# Mechanism: anxiety consumes central executive, reducing effective WM by ~1.5 items
has_math_anxiety = rng.random() < 0.20
anxiety_wm_reduction = rng.uniform(0.8, 1.5) if has_math_anxiety else 0.0
effective_wm = max(1.5, wm_capacity - anxiety_wm_reduction)

# Learning rate: correlated with WM (r ≈ 0.5)
# Sample as: learning_rate = base + wm_component
wm_z = (wm_capacity - 4.0) / 1.5  # standardize
schema_formation_rate = np.clip(rng.normal(0.3 + 0.1 * wm_z, 0.1), 0.05, 0.65)
ss_growth_rate = np.clip(rng.normal(1.0 + 0.15 * wm_z, 0.2), 0.3, 1.8)

# Forgetting rate: log-normal (FSRS empirical data)
forgetting_rate = np.clip(rng.lognormal(0.0, 0.35), 0.4, 2.5)

# Frustration threshold: inversely related to anxiety
frustration_threshold = np.clip(rng.normal(0.7 - 0.2 * has_math_anxiety, 0.1), 0.35, 0.9)
```

Also add a `metacognitive_bias` parameter (float, Normal(0, 0.2)) — positive = overconfident, negative = underconfident. Remove the current proxy that uses `boredom_threshold` for overconfidence detection in `_generate_confidence()`.

## Fix 4 — Enable Confidence Reports as a 4th Observation Modality

**Files:** `simulation/simulated_agent.py`, `active_inference/pomdp.py`

**Problem:** `Observation.confidence_report` is generated by `_generate_confidence()` but `prompt_confidence=False` everywhere in the simulation. Confidence is the key discriminator between the anxious archetype (underconfident) and overconfident archetype — without it, these two learner types are nearly indistinguishable from the other three observables.

**Fix part A — Simulated agent:**
- Set `prompt_confidence=True` in `simulation/monte_carlo.py` `run_session()`
- Fix `_generate_confidence()` to use the new `metacognitive_bias` parameter instead of the `boredom_threshold` proxy

**Fix part B — POMDP A matrix:**
- Add confidence as a 4th observation modality: 3 bins (low=1-2, medium=3, high=4-5)
- Build `_build_confidence_A()`: `P(confidence | RS, schema, metacalib_bias)`
  - High RS + full schema + positive bias → high confidence
  - Low RS + no schema + negative bias → low confidence
  - Overconfident: reports high even when RS is low
- Add to `NUM_OBS`, `A_DEPENDENCIES`, and `build_A_matrices()`

## Fix 5 — Fix the WM Overcorrection

**Files:** `meta_function.py`, `simulation/simulated_agent.py`

**Problem:** The ablation shows removing WM from policy decisions *improves* mastery by +0.044. The system is spending too many problems on easy material via `reduce_load`. The issue: high WM load from appropriately challenging problems (germane load) is being treated the same as high WM load from poor design (extraneous load).

**Fix:** In `meta_function.py` `select_action()`, modify Priority 3 (Don't overload):

```python
# Current: any high WM → reduce_load
if wm_label == "high":
    return ActionSelection(action="reduce_load", ...)

# Fixed: only reduce if accuracy is also falling (load is extraneous, not germane)
# Add recent_accuracy parameter to select_action()
if wm_label == "high" and recent_accuracy < 0.4:
    return ActionSelection(action="reduce_load", ...)
# If WM is high but accuracy is OK, the load is productive — don't reduce it
```

Pass `recent_accuracy` (rolling last-5-problems accuracy per skill) from the policy into `select_action()`. Track this in `_policy_meta_function()` using the observations already returned by `run_session()`.

## Fix 6 — Wire Affect into the A Matrix (Ashcraft Mechanism)

**Files:** `active_inference/pomdp.py`

**Problem:** Affect is a state factor but it doesn't change the likelihood of observations in a mechanistically grounded way. The ablation shows it contributes nothing to policy outcomes.

**Fix:** In `_build_accuracy_A()`, the affect modifier currently is:
```python
p += [-0.1, 0.0, -0.05][affect_i]  # frustrated, engaged, bored
```

Replace with the Ashcraft mechanism — anxiety (frustrated state) reduces *effective WM* rather than directly reducing accuracy:
```python
# Frustration compresses WM headroom — treat it as WM×1.25 for penalty purposes
# The penalty hits harder when WM is already moderate/high
effective_wm_load = wm_i + (1 if affect_i == 0 else 0)  # frustrated adds ~1 WM bin
effective_wm_load = min(effective_wm_load, WM_BINS["n"] - 1)
# Then apply WM penalty using effective_wm_load instead of wm_i
```

This means frustration matters most when WM is already at moderate/high — which matches the empirical finding that high-WM students are *more* impaired by anxiety (they rely more on WM-intensive strategies).

## Fix 7 — Make Schema Continuous in the Generative Process

**Files:** `simulation/simulated_agent.py`, `simulation/learner_types.py`

**Problem:** `GroundTruthState.schema` is discrete (0/1/2). Schema formation in `_update_schema()` fires probabilistically. The expertise reversal effect is graded — worked examples should become continuously less effective as schema grows, not flip at a threshold.

**Fix:**
- Change `schema` in `GroundTruthState` to a float [0.0, 1.0] (0=none, 0.5=partial, 1.0=full)
- Change `initial_schema` in `CognitiveParams` to float
- Update `_update_schema()`: increment by a small continuous amount per problem, modulated by action effectiveness and current schema level
  - Worked example effect decays: `schema_gain × (1 - schema)^0.5` — gain is largest at schema=0, approaches 0 at schema=1 (expertise reversal)
- Update `_determine_correct()` to use continuous schema value
- The POMDP state space keeps 3 discrete bins (none/partial/full) as its discretization of the continuous ground truth — that's fine, discretization is the POMDP's approximation

## Fix 8 — Scale the Monte Carlo to Population-Level

**Files:** `simulation/monte_carlo.py`, `simulation/learner_types.py`

**Problem:** `run_monte_carlo()` defaults to 8 archetypes × 10 seeds = 80 trajectories. The whole point of synthetic simulation is unlimited scale without IRB. The policy comparisons need statistical stability.

**Fix:** Add a `run_population_monte_carlo()` function that:
1. Samples `n_students` (default 500) learner types using the fixed `sample_learner_type()`
2. Runs each sampled student through all policies (or a subset)
3. Weights results by empirical prevalence (use a rough prior: skewed toward novice/partial_knowledge)
4. Reports bootstrap confidence intervals on policy rankings (not just point estimates)

The existing `run_monte_carlo()` with archetypes stays for fast development iteration. The population version is for research claims.

```python
# worked_example schema shift:

# Bjork: re-exposure without retrieval effort → minimal stability gain

```

The main change: add a comment block to each transition function citing the specific paper and effect size it's calibrated to. Makes it auditable.
